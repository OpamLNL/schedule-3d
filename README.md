# 3D Розклад коледжу

Інтерактивний веб‑застосунок для **планування навчального процесу** та **3D‑візуалізації розкладу** (React + Vite + Three.js).

Дипломний проєкт: навігація та планування навчального процесу коледжу.

## Можливості

- **Навплан** — імпорт Excel, нормалізація, редагування, класифікація рядків (пари / КР / практики…)
- **Версії навплану** — збереження змін у браузері (IndexedDB), завантаження без Excel, журнал «хто / коли»
- **Групи, аудиторії, преференції** — правила для автозбірки
- **Розклад (весна)** — автоматичне розміщення пар з урахуванням змін, поділів, накладок
- **3D циліндр** — фільтри, накладки, поділи, тижневий розклад викладача / групи
- **Збереження розкладів** — версії для подальшої роботи, друк по групах і викладачах

## Швидкий старт

```bash
cd schedule-3d
yarn
yarn dev
```

Відкрийте `http://localhost:5173` (або адресу від Vite).

### Опційно: демо‑дані з Excel

Excel‑файл навплану покладіть у **батьківську** папку проєкту, потім:

```bash
yarn import:xlsx
```

Скрипт оновить `src/data/schedule.json` для демо‑перегляду 3D (без повного навплану в UI).

### Збірка для продакшену

```bash
yarn build
yarn preview
```

## Робочий процес

1. **Навплан** — завантажити Excel або відновити збережену версію → відредагувати → **Зберегти версію**
2. **Групи / Аудиторії / Преференції** — перевірити налаштування
3. **Розклад** — **Зібрати весняний розклад** → **Зберегти версію**
4. **3D циліндр** — фільтри, перегляд накладок, **Зберегти для друку**

Дані зберігаються локально в браузері (`localStorage` + `IndexedDB`). Після F5 фільтри циліндра та активна вкладка (`#/cylinder`) відновлюються.

## Стек

- React 18, TypeScript, Vite
- Three.js, `@react-three/fiber`, `@react-three/drei`
- `xlsx` — імпорт навплану

## Структура

```
src/
  pages/           — вкладки (Навплан, Розклад, 3D циліндр…)
  scheduler/       — автозбірка весняного розкладу
  storage/         — localStorage, IndexedDB, версії документів
  components/      — UI та 3D-сцена
  utils/           — парсинг, фільтри, накладки, друк
scripts/
  import-xlsx.mjs  — CLI-імпорт у schedule.json
```

## Публікація на GitHub

Репозиторій: **https://github.com/OpamLNL/schedule-3d**

```bash
cd schedule-3d
git add .
git commit -m "Оновлення застосунку"
git push origin main
```

> Не комітьте `.env`, `node_modules/`, `dist/` та `.vercel/` — вони в `.gitignore`.

## Деплой на Vercel (з GitHub)

1. Зайдіть на [vercel.com](https://vercel.com) → **Add New… → Project**.
2. **Import** репозиторій `OpamLNL/schedule-3d` (GitHub має бути підключений до Vercel).
3. Налаштування (зазвичай підтягуються з `vercel.json` автоматично):

   | Поле | Значення |
   |------|----------|
   | Framework Preset | Vite |
   | Root Directory | `.` (корінь репо) |
   | Build Command | `yarn build` |
   | Output Directory | `dist` |
   | Install Command | `yarn install --frozen-lockfile` |

4. **Deploy** — отримаєте URL на кшталт `https://schedule-3d.vercel.app`.

Кожен `git push` у `main` автоматично перезбирає сайт. Змінні оточення на Vercel **не потрібні**.

### Що працює на хостингу

- Увесь UI: навплан, розклад, 3D, друк, імпорт/експорт JSON
- Дані **лише в браузері** користувача (localStorage + IndexedDB) — не на сервері Vercel
- Навігація через hash: `https://ваш-домен/#/cylinder`, `#/schedule` тощо
- «Обрати папку» для пакетного експорту — у Chrome/Edge (HTTPS на Vercel підходить)

### Локальна перевірка перед push

```bash
yarn build
yarn preview
```
